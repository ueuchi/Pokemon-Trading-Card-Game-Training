# デッキビルダー機能の実装

## 概要

作成したカードを使って60枚のデッキを作成できる機能を実装しました。

---

## 実装した機能

### ✅ デッキ作成
- デッキ名と説明を入力
- 60枚ちょうどのデッキを作成
- デフォルトカード + カスタムカードから選択

### ✅ カード管理
- カードを1枚ずつ追加/削除
- 同じカードは最大4枚まで
- リアルタイムで枚数カウント

### ✅ バリデーション
- **60枚ちょうどでないと保存不可**
- 同じカードが4枚を超えると追加不可
- ポケモンカードが最低1枚必要

### ✅ デッキ管理
- 複数のデッキを保存
- デッキの編集・削除
- ローカルストレージに自動保存

## 使い方

### 1. デッキビルダーを開く

画面上部のナビゲーションから「デッキ作成」をクリック

### 2. 新しいデッキを作成

1. 「新しいデッキを作成」ボタンをクリック
2. デッキ名を入力（例: 電撃デッキ）
3. 説明を入力（任意）
4. 「作成」ボタンをクリック

### 3. カードを追加

#### 左側のカードリストから選択

- **フィルター**: すべて / ポケモン / エネルギー
- **検索**: カード名で検索
- **追加**: 「+」ボタンでデッキに追加

#### 右側に現在のデッキが表示

- 追加したカードが表示される
- 「-」ボタンで1枚削除
- 「+」ボタンで1枚追加

### 4. デッキを保存

- **60枚ちょうどになると保存可能**
- 「デッキを保存」ボタンをクリック
- 60枚未満/超過の場合は保存不可

## デッキの制約

### 📋 DECK_CONSTRAINTS

```typescript
{
  TOTAL_CARDS: 60,      // デッキの合計枚数（必須）
  MIN_CARDS: 60,        // 最小枚数
  MAX_CARDS: 60,        // 最大枚数
  MAX_SAME_CARD: 4,     // 同じカードの最大枚数
  MIN_POKEMON: 1        // 最低限必要なポケモンの枚数
}
```

### バリデーションルール

1. **合計枚数**: ちょうど60枚
2. **同じカード**: 最大4枚まで
3. **ポケモン**: 最低1枚必要
4. **警告**: エネルギーが15枚未満または30枚超過

## データの永続化

### ローカルストレージ

デッキは自動的にブラウザのローカルストレージに保存されます。

**保存場所**: `localStorage['decks']`

**データ形式**:
```json
[
  {
    "id": "deck-1234567890-abc",
    "name": "電撃デッキ",
    "description": "でんきタイプ中心のデッキ",
    "cards": [
      {
        "card": { "id": "pikachu-001", "name": "ピカチュウ", ... },
        "count": 3
      },
      {
        "card": { "id": "energy-electric", "name": "でんきエネルギー", ... },
        "count": 20
      }
    ],
    "createdAt": "2026-02-11T...",
    "updatedAt": "2026-02-11T..."
  }
]
```

## 主要な機能

### DeckBuilderService

**役割**: デッキの作成・保存・管理

**主要メソッド**:

```typescript
// デッキを作成
createDeck(name: string, description?: string): Deck

// デッキを保存（60枚チェック）
saveDeck(deck: Deck): boolean

// デッキを削除
deleteDeck(deckId: string): void

// カードを追加
addCardToDeck(deck: Deck, card: Card, count: number): boolean

// カードを削除
removeCardFromDeck(deck: Deck, cardId: string, count: number): void

// 合計枚数を取得
getTotalCardCount(deck: Deck): number

// バリデーション
validateDeck(deck: Deck): DeckValidation
```

## トラブルシューティング

### Q: デッキが保存できない
A: 60枚ちょうどになっているか確認してください。画面右上の枚数表示を確認。

### Q: カードが追加できない
A: 以下を確認してください：
- デッキが60枚に達していないか
- 同じカードが4枚に達していないか

### Q: 保存したデッキが消えた
A: ブラウザのローカルストレージをクリアすると消えます。プライベートブラウジングモードでは保存されません。
